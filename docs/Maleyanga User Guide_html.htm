Maleyanga User Guide

 

Maleyanga um sistema de gesto de concees de crditos. Ela permite o cadastro de clientes, processamento de cadastro de pedidos de crdito, gesto das concees de crdito, clculo de juros e registro de pagamentos dos crditos.

 

Login

Para aceder ao sistema deve ter um browser de internet e o link (hiperligao) fornecido pelo seu administrador do sistema, conforme a imagem abaixo (fig. 1), o utilizador deve fornecer ao sistema o nome (username) e a senha(password) devidamente registrados no sistema.

Caso queira com que o sistema entre automaticamente sem o fornecimento prvio das credenciais, dever selecionar a opo ON na opo Remember me, mas esta opo desaconselhada, pois se a sua conta de Windows caso acedido por mais de uma utilizador, ficar-se- sem saber quem de facto acedeu ao sistema.

Clientes

Nesta opo acedemos ao programa de gesto de clientes, onde podemos criar, modificar e eliminar clientes, de notar que quem tem permisso para eliminar os clientes somente o administrador, entretanto caso este cliente j tenha um crdito associado a ele o sistema no ir permitir a eliminao deste cliente antes de se eliminar os respetivos crditos.

Cadastro de novos clientes

Para o cadastro de novos clientes, selecione a opo Novo Cliente, no registro temos a primeira opo Entidade que representa o nome do projeto ou empresa que estar associado a este novo cliente.

No cadastro do cliente o sistema permite anexar a cada cliente no mximo 10 em cada registro. Podendo adicionar mais anexos caso hajam.

Portanto antes do registro de clientes, temos que ter j registado a instituio concessionria de crdito. Para adicionar anexo click no boto ++ Anexos, que ir abrir 10 campos para o upload dos documentos. O sistema no tem opo para a eliminao dos anexos, portanto deve haver muito cuidado nos Upload dos documentos, tambm deve se evitar carregar documentos com mais de 10Mb de espao.

Classificao dos clientes:

         Clientes excelentes Lista de clientes que numa tiveram pagamento em atraso.

         Clientes booms Aqueles clientes que apesar de alguma vez terem tido prestaes atrasadas conseguiram honrar com seus compromissos.

Cadastro Do Novo Assinante (Avalista)

Pedidos de Crdito

 

Fig. 6 Programa de gesto de clientes.

O programa de Pedidos de crdito, permite a gesto de pedidos de crditos pelos clientes, ele por si s no constitui garantia e nem condio para o a cadastro de crdito.

Este programa permite que seja cadastrado todos os pedidos de crdito.

Cadastro de Novo Pedido de crdito (PDC)

Para cadastrar o pedido de crdito selecionar a opo E preencha o respetivo formulrio, de notar que o Valor da Penhora deve ser 2 vezes superior o Valor de Crdito.

A opo Estado tem 4 variveis, representado quatro possveis estado dos pedidos de crdito, nomeadamente:

1.      Aberto todos os pedidos de crdito, no incio ficam no estado de aberto.

2.      Pendente Quando o PDC foi ainda aprovado por falta de algum requisito.

3.      Aprovado - caso o PDC tenham chumbado na avaliao.

4.      Reprovado Caso o pedido tenha sido aprovado

Crditos

No programa de crditos temos a gestao dos crditos gerados. Os crditos podem ser gerados sem os PDC.

Cadastro de novo crdito

Para criar novo cadastro de crdito selecione a opo , Onde ir selecionar o cliente responsvel pelo crdito, (caso se trate de um novo cliente, dever cadastra-lo antes); dever lanar todos os campos e a seguir selecione a opo Create.

 

Fig-9. Formulrio de cadastro de crdito

         Valor Creditado(PV) Valor creditado incluindo taxa e sem juros.

Percentual de Juros(rate) percentual de juros, dever digitar o percentual de juros, somente valores de 1 a 100, se por exemplo os juros do valor creditado for de 40%, neste campo iremos digitar somente o valor da percentagem que 40.

A opo Valor Creditado com Juros ativado caso o valor preenchido no campo Valor Creditado j inclua os respetivos juros. Entretanto desaconselhamos a aplicao desta opo.

A opo Numero De Prestaes corresponde ao nmero de prestaes que a dvida ser desembolsada e cada prestao tem a durao de 30 dias, o valor frequentemente usado neste campo o valor 1. Que representa uma nica prestao a ser paga em 30 dias.

O campo Data De Conceo de Credito deve se selecionar clicando o cone de calendrio ao lado e escolher a devida data e no preencher manualmente no campo. Ter em ateno que a data de conceo de crdito deve corresponder de facto ao dia que o cliente recebeu o crdito, pois o sistema ir usar esta data a gerao das prestaes de crdito com as respetivas datas previstas para o pagamento.

Abaixo esta o processo de crdito, o crdito s dado com o estado Fechado caso tenham sido pagas todas as prestaes geradas pelo sistema e os devidos juros de demora. E essse processo ocorre automaticamente logo que se verifique que todas as prestaes foram pagas.

Aps o cadastro de um novo crdito o sistema mostra os dados lanados e a listagem das respetivas prestaes. Podendo selecionar cada prestao visualizar a prestao em detalhes.

Prestaes

As prestaes so gerados automaticamente pelo sistema quando se faz o cadastro de um novo crdito.

Lanamento de pagamentos das prestaes.

O sistema permite o lanamento de vrios pagamentos correspondentes a uma prestao.

Para lanar um novo pagamento selecione o crdito em causa, no campo Prestaes tem listado todas as prestaes a serem pagas, selecione a prestao que pretende pagar, a seguir no submenu selecione a opo Novo Pagamento, no campo Valor Pago digite o valor pago e selecione no campo Data de Pagamento a data que o pagamento foi efetivado, na campo Descrio pode lanar dados relativos ao pagamento. selecione a opo Choose File apara anexar o comprovativo de pagamento digitalizado, e clica no boto Create, o sistema ir fazer o processamento de pagamento lanado, caso o valor em dvida seja superior ao valor pago, o sistema manter a prestao como estando em dvida, at que sejam lanados o valor remanescente com novos o respetivos juros de mora caso tenha ultrapassado a data prevista para o pagamento.

 

Lanamento de remisses

O sistema tambm permite a lanamento de valores de remisso, coso o cliente tenha sido beneficiado do perdo da parte de sua dvida. Entretanto esta operao s permitido aos Administradores do sistema. O Processo de lanamento das remisses semelhante ao processo de lanamento de pagamentos.

 

Gesto de Utilizadores

Os administradores do sistema podem cadastrar novos utilizadores do sistema e atribuir-lhes os respetivos papais.

 

Cadastro de novo Utilizador do sistema

O link para aceder a gestao de Utilizadores encontra-se no sub-menu Settings, localizado na menu preto do cabealho da aplicao do lado direito.

Para cadastrar um novo utilizador do sistema, selecione a opo Novo Utilizador. Logo a seguir o sistema ir mostrar o formulrio para o cadastro de novo utilizador. Selecione o usernane (Nome do utilizador) e a respetiva senha (password). Outros campos poder deixa-los na opo padro (default). Aps o cadastro do cliente dever recuar a submenu anterior para definir o perfil do novo utilizador.

Definio do Perfil do novo utilizador.

Aps o cadastro do novo utilizador dever se atribuir o respetivo perfil. De notar que a opo Definir Perfil do Novo(a) Utilizador, irreversvel.

 

Taxas

O programa Taxas, localizado no Sub-menu Settings permite o laamento de taxas a serem aplicados sobre os crditos.

Lanamento de nova Taxa

Para a criao da nova Taxa selecione a opo Novo(a) Taxa, Nom campo Nome, crie uma designao da taxa. O campo Valor, corresponde ao valor mnimo da Taxa, portanto bastar preencher este campo e logo a seguir o percetual de juros sobre o valor. Os campos Valor Mnimo e o Valor Maximo,selecione a opo ON no campo Activo caso queira que o sistema aplique a nova tava imediatamente. Por ltimo pressione o boto Create para salvar os dados.
